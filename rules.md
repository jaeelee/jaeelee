# rule
## 네이밍 규칙
### 1. project명 규칙

### 2. repository 규칙
- 특수문자, CamelCase 금지
- 소문자 사용
- 대시 사용
- 명확하게 작성
- 일관성있게 작성
- '프로젝트 이름-개발 환경-프로젝트 목적' 형태로 구성

### 3. branch 생성 규칙
- 특수문자, CamelCase 금지
- 소문자 사용
- 대시 사용
- 명확하게 작성
- 일관성있게 작성
- '이름-내용-이슈번호' 형태로 구성

### 4. ISSUE 네이밍 방법
- 키워드 중심으로 작성

### 5. PR 네이밍 방법
- 키워드 (프로젝트 시작 전 키워드 설정)
- 예시) SET - 환경세팅, MP - 메인페이지, TS - 텍스트 검색, IS - 이미지 검색
- '키워드 이슈번호-제목작성'
